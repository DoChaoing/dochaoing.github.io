RAG Complete Guide: Give Your LLM Private Knowledge

  • 2025-04-22
  • 作者 Chao
  • ~8.92K 字

TL;DR — LLMs are powerful but blind to your private data. RAG is the bridge. This guide covers everything from first principles to production. 15 min read.

LLMs are incredible — until you ask about your own data. RAG (Retrieval-Augmented Generation) fixes this by letting models look things up before answering. No fine-tuning required. Here's the complete playbook.

🧠 Why RAG?

The LLM Blind Spots

Limitation Impact Severity
📅 Knowledge cutoff Can't answer about recent events 🔴 Critical
🔒 Private data blind Zero access to internal docs 🔴 Critical
🎭 Hallucination risk May fabricate plausible-sounding answers 🟡 High

Solution Comparison

Approach How Pros Cons Rating
Fine-tuning Continue training on domain data Deep adaptation Expensive, slow to update ★★★☆☆
RAG Retrieve docs → feed to model Real-time, explainable Needs retrieval infra ★★★★★
Long Context Stuff everything into context Simple Expensive, has limits ★★★★☆

RAG is the industry standard — best balance of quality, cost, and maintainability.

🔑 Key Concepts

Embedding

Convert text → vector. Semantically similar text → nearby vectors.

from openai import OpenAI
client = OpenAI()

response = client.embeddings.create(
    model="text-embedding-3-small",
    input="Convert this text to a vector"
)
vector = response.data[0].embedding  # 1536 dimensions

2. Build a RAG System

Step 1: Document Processing

from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,       # Chunk size
    chunk_overlap=50,     # Overlap for context continuity
    separators=["\n\n", "\n", "。", " "]
)

chunks = splitter.split_text(document_text)

💡 Chunking is critical

  • Too large → imprecise retrieval
  • Too small → incomplete context
  • Sweet spot: 500-1000 chars with overlap

Step 2: Embed & Store

from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

vectorstore = Chroma.from_texts(
    texts=chunks,
    embedding=embeddings,
    persist_directory="./chroma_db"
)

Step 3: Retrieve & Generate

from langchain_openai import ChatOpenAI
from langchain.chains import RetrievalQA

llm = ChatOpenAI(model="gpt-4o")

qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",
    retriever=vectorstore.as_retriever(
        search_type="mmr",  # Maximal Marginal Relevance
        search_kwargs={"k": 5}
    )
)

result = qa_chain.invoke("What is RAG?")

3. Advanced Optimizations

🚀 Query Rewriting

User queries are often vague. Rewrite for clarity:

# Original
query = "how to use that thing"

# Rewritten
rewritten = "How to implement a RAG pipeline for document Q&A"

🚀 Hybrid Search

Combine vector + keyword search for best of both worlds:

# Vector search — semantic understanding
vector_results = vectorstore.similarity_search(query, k=10)

# BM25 keyword search — exact matching
keyword_results = bm25_search(query, k=10)

# Reciprocal Rank Fusion — merge & re-rank
final_results = reciprocal_rank_fusion(
    vector_results, keyword_results
)
Method Strength Weakness
Vector only Semantic understanding Weak exact match
Keyword only Exact match No semantic understanding
Hybrid Both More complex

🚀 Cross-Encoder Reranking

Fine-grained relevance scoring:

from sentence_transformers import CrossEncoder

reranker = CrossEncoder('BAAI/bge-reranker-large')
pairs = [(query, doc) for doc in candidates]
scores = reranker.predict(pairs)
ranked = sorted(zip(candidates, scores), key=lambda x: x[1], reverse=True)

🚀 Metadata Filtering

Precision filtering on document attributes:

results = vectorstore.similarity_search(
    query,
    filter={
        "category": "engineering",
        "year": 2025,
        "team": "platform"
    }
)

🔷 LangChain Quick Start

from langchain.indexes import VectorstoreIndexCreator
from langchain.document_loaders import TextLoader

loader = TextLoader("./data/knowledge.txt")
index = VectorstoreIndexCreator().from_loaders([loader])
result = index.query("Your question here")

🔶 LlamaIndex Quick Start

from llama_index.core import VectorStoreIndex, SimpleDirectoryReader

documents = SimpleDirectoryReader("./data").load_data()
index = VectorStoreIndex.from_documents(documents)
query_engine = index.as_query_engine()
response = query_engine.query("Your question here")

6. Troubleshooting Guide

❓ Irrelevant Retrieval

Root Cause Fix
Bad chunking Tune chunk_size / overlap
Weak embeddings Upgrade embedding model
Vector-only search Add hybrid search
Poor ranking Add cross-encoder reranking

❓ Hallucinated Answers

Fix: Ground the model explicitly:

prompt = """
Answer ONLY based on the retrieved context below.
If the context doesn't contain the answer, say "I don't know."

Context:
{context}

Question: {question}
"""

❓ Slow Response

Bottleneck Optimization
Slow retrieval Reduce k, use faster vector DB
Slow embedding Cache embeddings, use smaller model
Slow generation Use faster LLM, quantized models
Overall latency Cache frequent queries

7. Best Practices

Rule Why
Garbage in → Garbage out Document quality > algorithm optimization
Chunk wisely Different doc types need different strategies
Retrieve precisely Fewer, better results > many, noisy ones
Cite sources Let users verify answers
Iterate forever Collect feedback, improve continuously

Bottom line: RAG = Retrieve + Generate. Let the model "look it up" before answering. Start with LangChain, go deep with custom pipelines.
